HTML

  • The head element contains information about the webpage.
  • The head element contains the meta information about the webpage.
  • The meta information tells the browser information on how to display the page.
  • The meta information isn't displayed to the user.
  • The body element represents the visible content shown to the user.
  • The body element contains a header section, in which there is usually a "<'h1>"" tag.
  • There are 6 levels of headers from h1 to h6.
  • there should only be one h1 on any webpage.
  • "Main" resides within the body and contains the main body of the webpage.
  • "main" can be divided into section with the section tag.
  • Secitons can be tagged with identifiers such as class and id, which are useful later for css and maybe javascript.
  • Images can be inserted with ths syntax: img src="image source(url, or other i assume)" alt="word description" and this tag is ended simply by a slash at the end rather than using a typical end tag.
  • Most elements have opener and closer tags, For example, <'h1> and <'/h1>.
  • However, some elements like "img" close within the first set of chevrons by adding a '/' at the end of a single chevron set. i.e. '<'img src="./assets/bowtie-cat.png" alt="Profile image of cat wearing a bow tie." />
  • links to other websites can be added by using the "a" tag with syntax '<'a href="url">"Your text here" '<'/a>
  • Some elements of HTML are semantic, meaning their tag names indicate their role. For example "header" "footer "body"
  • Some elements are non-semantic. For example, "div" and "span"

CSS

  • CSS can be used inline, with an internal style sheet, or with and external style sheet.
  • Most development will be done with an external style sheet because of a concept called "Separation of Concerns."
  • Separation of Concerns is separating code into different files based on use. This must make it easier to determine code function and modify in the future.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • header and footer commands can change the width(in percentage), the height in pixels, the backgroun color and text color ,etc.
  • Text-align indicates where the text is to be placed, i.e. center.
  • Background color dictates the background color of respective section.
  • Display: block makes item a block like a paragraph, starting it on a new line.
  • Images can be resized using pixels.
  • Font-size can be used in pixels to adjust font size.
  • CSS can be applied to each section as defined by class, and I assume by id. These variables are defined in each "section" within the HTML.
  • Text-shadow adds a shadow to text.

Git

  • Git status = checks which branch we are in
  • Git checkout -b "branch name" = checkout move the working branch to a new branch, -b flag creates a new branch,"branch name" is the name of the new branch, and switches to new branch
  • Git add -A = add adds modifications in the current working branch to the staging area, -A indicates that we want to add all changes, The staging area informs Git that the changes will be added in the next commit
  • Git commit -m “add starter code” = commit creates a snapshot of the currently staged changes. Once committed, our working branch will be updated with the starter code.
  • Git pull origin main = pull commands to receive a branch’s modification into the local environment, origin indicates the source as the GitHub repo, main indicates the branch, also switches to main branch
  • Git push origin feature/starter code = push pushes changes from local to GitHub, origin indicates GitHub repo, feature/starter code is data name
